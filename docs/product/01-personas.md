## Persona 1: 小小探险家 - Primary Child User

她是小学阶段的孩子，有作业、阅读、背诵、整理等日常任务，但不希望被一个严肃的工具监督。她愿意为一个可爱的伙伴打开产品，但会对责备、失败暗示或复杂设置迅速失去兴趣。

### Persona Card

**小小探险家 - Primary Child User**  
她想看到自己的努力被马上看见，也想拥有一个只属于自己的小伙伴。她不需要复杂目标管理，只需要“今天做什么、做完有什么变化、宠物怎么回应我”。

**Key quote:** “我想看看星星今天会不会长大。”

**Goals.** 完成今天最重要的学习小事；获得温和鼓励；看到宠物和房间有变化；把习惯变成可期待的日常。

**Frustrations.** 被催促；任务太多；看不懂规则；完成后没有反馈；未完成时被暗示“让宠物难过”。

**Design rules - always.** 视觉第一、文字短；完成动作要有即时反馈；失败状态只给下一步。

**Design rules - never.** 不责备、不比较、不排名；不让宠物受伤或消失；不把学习包装成消费/抽奖压力。

### Jobs to Be Done

**Functional.** 当她放学后面对任务时，她需要知道先做哪一件小事，并能用一次轻松打卡记录完成。

**Emotional.** 当她完成任务时，她想感觉“我做到了，而且有人看见”，而不是“我只是交差”。

**Social.** 她希望家长看到自己的进步，但不希望每个细节都被审判。

**Underlying.** 她真正想要的是把学习任务从外部要求转化成“我和宠物一起变厉害”的关系。

### Product Decision Rules

- 轻量打卡优先于复杂任务管理。
- 正向成长优先于扣分/负反馈。
- 宠物状态优先于数字报表。
- 选择一个小目标优先于展示完整待办压力。
- 亲子可见优先于公开社交。

## Persona 2: 爸爸妈妈 - Parent Operator

家长想帮孩子建立学习习惯，但不想每天维护一个复杂系统，也不希望产品制造新的亲子冲突。家长需要设置任务、确认关键任务、看到趋势，并在必要时关闭 AI 或删除数据。

| --- | --- |

| Valid for | 家庭私用、单孩或少量孩子、家长主导任务 |
| Not valid for | 老师批量管理、学校系统、公开竞赛 |

### Persona Card

**爸爸妈妈 - Parent Operator**  
家长不是来“玩一个宠物游戏”的，而是来降低提醒成本、提高孩子自主完成概率。家长愿意配置系统，但每天只能给它很短时间。

**Key quote:** “我希望她愿意自己打开，而不是我每天又多一个要管的东西。”

**Goals.** 快速设置任务；确认关键完成；看出孩子是否更稳定；保护隐私；避免产品让孩子上瘾或焦虑。

**Frustrations.** 配置复杂；需要频繁确认；奖励过度游戏化；无法解释数据存哪；孩子为了奖励敷衍完成。

**Design rules - always.** 家长端每个任务可在 30 秒内设置；关键隐私开关清楚；趋势只展示可行动信息。

**Design rules - never.** 不做隐藏消费；不默认上传儿童数据；不诱导家长增加任务负担。

### Jobs to Be Done

**Functional.** 当家长想培养习惯时，需要把少数关键任务变成孩子能理解、能坚持的日常闭环。

**Emotional.** 家长想感觉“我在支持孩子”，而不是“我在用另一个系统控制孩子”。

**Social.** 家长希望孩子逐渐自主，也希望家庭沟通更轻松。

**Underlying.** 家长真正购买的是“亲子摩擦下降 + 习惯稳定上升”的组合结果。

### Product Decision Rules

- 默认配置优先于空白配置。
- 2 分钟家长维护上限优先于功能完整性。
- 本地/最小数据优先于云同步。
- 可解释规则优先于黑盒 AI。
- 家庭试用证据优先于上线幻想。
