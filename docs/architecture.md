# 网页架构说明

状态：Web 主线  
日期：2026-06-21  

## 核心模块

`src/lib/domain.ts` 是核心规则引擎，保持纯函数优先，方便 UI、单元测试和后续后端迁移共用。

它负责：

- 初始化家庭宠物状态。
- 生成每日固定、一次任务、本周目标。
- 处理打卡、家长确认、撤销。
- 计算每日能量上限、总能量、成长阶段和连续成长。
- 生成宠物状态快照和安全文案。
- 添加、更新、排序、停用任务模板。
- 导出和重置家庭数据。

`src/lib/storage.ts` 是浏览器本地存储适配层，负责把规则状态保存到 `localStorage`。第一版不需要账号、后端或云数据库。

`src/App.tsx` 是产品主界面，包含孩子端宠物房间、今日任务、家长中心、宠物日记和隐私页。

`src/components/PetIllustration.tsx` 是当前 MVP 的网页宠物视觉资产。它使用 SVG 和 CSS 动效，保证部署包可控；后续可以替换为 `hatch-pet` 生成的 spritesheet。

## 数据模型

- `ChildProfile`：昵称、年龄段、创建时间。
- `PetProfile`：宠物名、物种、总能量、阶段、解锁物、最近互动。
- `TaskTemplate`：任务模板，支持 daily、once、weekly。
- `DailyTask`：某日或某周生成的可打卡任务。
- `CheckIn`：孩子提交的打卡记录。
- `EnergyLedger`：实际获得的成长能量流水。
- `PetStateSnapshot`：每日宠物状态快照。
- `ParentSettings`：家长控制和隐私设置。
- `DiaryEntry`：规则化文案日志。

## 能量规则

- 每日成长能量上限为 40。
- 需要家长确认的任务在确认前不发放能量。
- 每个任务只能发放一次能量。
- 未完成任务不会产生负能量、扣血或坏状态。
- 成长阶段按累计能量解锁：0、30、90、180。

## 页面流

- 宠物房间：展示宠物、今日任务、打卡和互动。
- 家长中心：管理资料、任务、确认、趋势、导出和删除。
- 宠物日记：生成和查看规则化安全文案。
- 隐私：展示本地优先、最小化收集、导出和重置入口。

## 发布边界

当前正式路线是网页部署，不再以微信小程序二维码作为验收目标。`miniprogram/` 和 `cloudfunctions/` 目录不参与当前构建、测试和部署。
